### 需求：迷你控件在紧凑模式下显示

当计时器窗口处于紧凑模式（任一维度 < 300px）时，系统应当在计时器表盘下方显示一排纤细的三个纯图标按钮：重启当前轮次、播放/暂停和跳过轮次。这些控件应当以固定大小渲染，不受表盘缩放比例影响。

#### 场景：紧凑模式下控件出现

- **当** 窗口宽度或高度降到 300px 以下
- **则** 重启、播放/暂停和跳过按钮应当在表盘下方可见

#### 场景：正常模式下控件不存在

- **当** 窗口在两个维度上都在紧凑阈值之上
- **则** 迷你控件不应渲染（改为显示完整控件行）

#### 场景：播放/暂停图标反映运行状态

- **当** 计时器正在运行
- **则** 播放/暂停按钮应当显示暂停图标
- **当** 计时器已暂停或空闲
- **则** 播放/暂停按钮应当显示播放图标

#### 场景：重启按钮重启当前轮次

- **当** 用户在紧凑模式下点击重启按钮
- **则** 当前轮次的已用时间应当重置为零，不推进序列

#### 场景：跳过按钮推进到下一轮次

- **当** 用户在紧凑模式下点击跳过按钮
- **则** 计时器应当推进到序列中的下一个轮次

#### 场景：控件独立于表盘缩放

- **当** 表盘缩放比例处于最小值（0.4）
- **则** 迷你控件按钮仍应以其完整定义尺寸（24×24px）渲染
