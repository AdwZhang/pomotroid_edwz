### 需求：统计窗口可从主标题栏访问

系统应当在主窗口标题栏提供一个图表/统计图标按钮。激活后应当打开一个专用的统计窗口（Tauri `WebviewWindow`，标签 `"stats"`，路由 `/stats`）。如果窗口已经打开，应当聚焦而非重复打开。窗口应当不可调整大小，在 Linux/Windows 上无装饰（与设置窗口行为一致），尺寸约为 820x540。

#### 场景：首次打开统计窗口

- **当** 用户点击主标题栏的统计图标
- **则** 在 `/stats` 路由打开一个新的统计窗口

#### 场景：统计窗口已打开

- **当** 用户在统计窗口已打开时点击统计图标
- **则** 聚焦现有窗口（不创建重复窗口）

#### 场景：主题应用到统计窗口

- **当** 统计窗口打开
- **则** 通过 CSS 自定义属性应用与主窗口相同的活跃主题

---

### 需求：统计窗口内的三标签导航

统计窗口应当显示三个标签：**今天**、**本周**和**全部时间**。默认活跃标签应当为**今天**。切换标签应当加载对应的数据视图，无需重新打开窗口。

#### 场景：打开时的默认标签

- **当** 统计窗口首次打开
- **则** "今天"标签处于活跃状态并显示今天的数据

#### 场景：切换标签

- **当** 用户点击一个标签
- **则** 渲染对应的视图及其数据

---

### 需求：今天标签 — 统计卡片

"今天"标签应当显示三个摘要统计卡片：

1. **轮次** — 今天（本地日期）已完成的工作会话计数
2. **专注时间** — 今天已完成工作会话的总分钟数，格式为 `X小时 Y分钟`（时和分）
3. **完成率** — 今天已启动工作会话中完成的百分比，显示为 `N%`

#### 场景：有会话的日子

- **当** 用户今天有已完成的工作会话
- **则** 三个统计卡片反映今天的准确计数

#### 场景：今天无会话

- **当** 今天没有记录任何会话
- **则** 卡片显示 `0` / `0小时 0分钟` / `—`（无会话启动时完成率显示为破折号）

---

### 需求：今天标签 — 逐小时时间线

"今天"标签应当以柱状图形式显示 24 小时中每小时的会话分布，柱高代表该小时内完成的工作轮次数。无会话的小时应当渲染为最小基线高度（而非不可见的零高度柱）。

#### 场景：不同小时有会话

- **当** 用户在一天中多个小时有会话
- **则** 会话更多的小时柱形更高

#### 场景：空白日

- **当** 今天不存在任何会话
- **则** 所有柱形以最小高度显示，附带空状态标签

---

### 需求：本周标签 — 每日柱状图

"本周"标签应当显示过去 7 个日历天（本地日期）每天已完成工作轮次的柱状图，带有星期标签（周一至周日或等效标签）。今天的柱形应当有视觉区分（如强调色或高亮）。

#### 场景：活动混合的一周

- **当** 某些天有会话而其他天没有
- **则** 柱形反映每天完成的轮次数；空白日显示零高度柱形

#### 场景：今天的柱形

- **当** 查看周图表
- **则** 今天的柱形与过去几天在视觉上有所区分

---

### 需求：本周标签 — 连续天数计数器

"本周"标签应当显示**当前连续天数**：截至今天或昨天，用户连续完成至少一个工作会话的日历天数。在当前日午夜之前连续天数应当保持活跃 — 如果昨天有会话但今天还没有，连续天数仍然显示为活跃。

#### 场景：包含昨天的活跃连续天数

- **当** 用户昨天完成了会话但今天还没有
- **则** 连续天数计数器显示包含昨天的连续天数（连续记录仍然有效）

#### 场景：包含今天的活跃连续天数

- **当** 用户今天已完成至少一个会话
- **则** 今天计入连续天数

#### 场景：连续天数中断

- **当** 今天和昨天都没有完成会话
- **则** 连续天数计数器显示 0

---

### 需求：全部时间标签 — 年度热力图

"全部时间"标签应当显示一个热力图网格（GitHub 贡献图风格），每个单元格代表一个日历日，可按年导航。单元格颜色强度应当使用固定的 4 级刻度反映当天完成的工作轮次数：0 轮次（背景色）、1-3（级别 1）、4-7（级别 2）、8+（级别 3）。颜色应当从活跃主题的 `--color-focus-round` 以不同透明度混合到 `--color-background` 派生。

#### 场景：高轮次数的日子

- **当** 某天有 8 个或更多已完成的工作轮次
- **则** 其单元格以最大强度（级别 3）显示

#### 场景：无轮次的日子

- **当** 某天没有已完成的工作轮次
- **则** 其单元格以背景强度（级别 0，视觉上为空）显示

#### 场景：统计窗口打开时主题变更

- **当** 统计窗口打开时活跃主题发生变更
- **则** 热力图单元格颜色更新以反映新主题颜色

---

### 需求：全部时间标签 — 终身总计

"全部时间"标签应当显示终身总计统计：总已完成工作轮次、总专注时间（格式为小时）和有史以来最长连续天数。

#### 场景：全部时间数据显示

- **当** 用户打开"全部时间"标签
- **则** 终身总计从完整的会话表中准确派生并显示

#### 场景：首次启动（空数据库）

- **当** 数据库中不存在任何会话
- **则** 所有总计显示零值，附带适当的空状态消息

---

### 需求：统计窗口响应计时器事件

统计窗口在打开时，应当在计时器轮次完成时实时更新其数据视图。"今天"和"本周"标签应当立即刷新；"全部时间"标签仅在之前已加载时才刷新。

#### 场景：统计窗口打开时轮次完成

- **当** 一个计时器轮次完成且统计窗口处于打开状态
- **则** "今天"和"本周"视图更新以反映新记录的会话，无需关闭并重新打开窗口

---

### 需求：今天标签 — 会话列表（含任务备注）

"今天"标签应当在小时柱状图下方显示今日所有 work session 的列表。每行显示时间段（HH:MM – HH:MM）和任务备注文本。无备注的 session 显示灰色斜体"无描述"。列表按 `started_at` 升序排列。

数据通过 `stats_get_detailed` 命令的 `DailyStats.sessions` 字段返回，类型为 `Vec<SessionEntry>`，其中 `SessionEntry` 包含 `started_at`、`ended_at`、`duration_secs`、`completed`、`task_note` 字段。

#### 场景：有会话时显示列表

- **当** 今天有 work session 记录
- **则** 小时柱状图下方显示会话列表，每行含时间段和备注

#### 场景：无会话时不显示列表

- **当** 今天没有任何 work session
- **则** 会话列表区域不渲染
