## 目的

定义番茄钟工作轮次的任务备注功能——用户可以记录当前正在做什么，备注随完成的 session 存入数据库，方便回看。

## 需求

### 需求：任务备注设置开关

系统应当在 `Settings` 结构体中提供 `task_notes_enabled: bool` 字段（DB 键：`task_notes_enabled`，默认 `false`）。开启后主计时器界面显示任务备注输入框，关闭时隐藏。

#### 场景：默认关闭

- **当** 新用户首次启动应用
- **则** `task_notes_enabled` 为 `false`，主界面不显示任务备注输入框

#### 场景：开启后显示输入框

- **当** 用户在设置 → 计时器中开启"任务备注"
- **则** 主界面在 work 轮次时显示任务备注输入框

---

### 需求：当前任务备注持久化

系统应当在 `Settings` 结构体中提供 `current_task_note: String` 字段（DB 键：`current_task_note`，默认 `""`，最大 100 字符）。该值跨轮次保留，直到用户手动修改或清空。

#### 场景：备注跨轮次保留

- **当** 用户在第 1 轮输入 "写文档" 并完成该轮进入休息再开始第 2 轮
- **则** 第 2 轮的备注文本仍为 "写文档"

#### 场景：备注跨应用重启保留

- **当** 用户设置备注后关闭并重新打开应用
- **则** 备注文本保持不变

---

### 需求：任务备注输入框 UI

系统应当在主计时器界面的 round label 下方、controls 上方显示单行文本输入框。输入框需满足：

- 最大长度：100 字符
- 居中显示，max-width 200px
- 仅在 `task_notes_enabled == true && round_type == "work"` 时可见
- 修改时 300ms debounce 后保存到设置
- **视觉状态区分：**
  - 默认（未聚焦）：透明背景、无边框，看起来像文本标签
  - 悬停：出现浅色背景
  - 聚焦（编辑中）：显示边框 + 背景，明确的输入框形态

#### 场景：输入框仅在 work 轮次显示

- **当** 当前轮次为 short-break 或 long-break
- **则** 任务备注输入框不可见

#### 场景：编辑后自动保存

- **当** 用户修改备注文本并停止输入 300ms
- **则** 新值通过 `settings_set("current_task_note", value)` 保存到数据库

---

### 需求：Compact 模式下的任务备注

在 compact 模式（窗口 <300px）下，任务备注应当以只读标签形式显示，附带一个小笔编辑按钮。点击编辑按钮后切换为输入框模式，失焦或按 Enter/Escape 退回只读模式。

#### 场景：compact 模式只读显示

- **当** 窗口进入 compact 模式且有备注文本
- **则** 备注以居中文本标签显示，右侧有编辑图标按钮

#### 场景：compact 模式无备注

- **当** 窗口进入 compact 模式且备注为空
- **则** 显示灰色斜体 placeholder 文本 + 编辑按钮

#### 场景：compact 模式编辑

- **当** 用户在 compact 模式点击编辑按钮
- **则** 标签变为输入框并自动聚焦
- **当** 用户按 Enter、Escape 或输入框失焦
- **则** 退回只读标签模式

---

### 需求：Session 完成时记录任务备注

数据库 `sessions` 表应当包含 `task_note TEXT`（可空）列。当 work 轮次完成（自然完成或 skip）时，系统应当读取当前 `current_task_note` 的值并写入该 session 行的 `task_note` 字段。休息轮次的 `task_note` 始终为 NULL。

#### 场景：work 轮次完成时记录备注

- **当** 一个 work session 自然完成，且 `current_task_note` 为 "写周报"
- **则** 该 session 行的 `task_note` 为 "写周报"

#### 场景：运行中修改备注取完成时的值

- **当** 用户在 work session 运行中将备注从 "A" 改为 "B"
- **则** session 完成时记录 "B"（取完成时的最新值）

#### 场景：空备注记录为 NULL

- **当** `current_task_note` 为空字符串
- **则** session 行的 `task_note` 为 NULL

#### 场景：休息轮次不记录备注

- **当** 一个 short-break 或 long-break session 完成
- **则** 该 session 行的 `task_note` 为 NULL

---

### 需求：日视图显示带备注的会话列表

统计窗口"今天"标签应当在小时柱状图下方显示今日所有 work session 的列表。每行包含：
- 时间段（HH:MM – HH:MM 格式）
- 任务备注文本（无备注时显示灰色斜体"无描述"）

列表按 `started_at` 升序排列，仅包含 `round_type = 'work'` 的 session。

#### 场景：有备注的会话显示

- **当** 今天有一个 09:30-10:05 的 work session，备注为 "Code Review"
- **则** 列表中显示 "09:30 – 10:05  Code Review"

#### 场景：无备注的会话显示

- **当** 今天有一个 14:00-14:25 的 work session，备注为 NULL
- **则** 列表中显示 "14:00 – 14:25  无描述"（灰色斜体）

#### 场景：无会话时不显示列表

- **当** 今天没有任何 work session
- **则** 会话列表区域不渲染
