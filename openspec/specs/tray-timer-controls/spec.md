### 需求：托盘菜单暴露计时器控制操作

托盘上下文菜单应当包含三个计时器控制项 — 切换（开始/暂停/恢复）、跳过和重置轮次 — 位于现有的显示和退出项之上。

#### 场景：托盘创建时菜单项存在

- **当** 托盘图标被创建
- **则** 菜单应当包含 id 为 "toggle"、"skip" 和 "reset-round" 的项

#### 场景：计时器控制项出现在显示和退出之上

- **当** 用户打开托盘上下文菜单
- **则** 切换、跳过和重置轮次项应当出现在分隔符、显示和退出项之前

### 需求：切换项标签反映当前计时器状态

切换项在计时器空闲时应当显示"开始"，运行时显示"暂停"，已暂停时显示"恢复"。

#### 场景：初始托盘创建时标签为"开始"

- **当** 托盘图标首次创建且计时器空闲
- **则** 切换项标签应当为"Start"

#### 场景：计时器启动时标签变为"暂停"

- **当** 计时器从空闲或已暂停转为运行
- **则** 切换项标签应当更新为"Pause"

#### 场景：计时器暂停时标签变为"恢复"

- **当** 计时器被暂停
- **则** 切换项标签应当更新为"Resume"

#### 场景：计时器重置时标签恢复为"开始"

- **当** 计时器被重置为空闲
- **则** 切换项标签应当更新为"Start"

### 需求：计时器空闲时跳过和重置轮次被禁用

跳过和重置轮次仅在计时器运行或暂停时才应启用。

#### 场景：初始托盘创建时跳过和重置轮次被禁用

- **当** 托盘图标首次创建且计时器空闲
- **则** 跳过和重置轮次项应当被禁用（不可交互）

#### 场景：计时器运行时跳过和重置轮次启用

- **当** 计时器转为运行
- **则** 跳过和重置轮次项应当变为启用

#### 场景：计时器暂停时跳过和重置轮次启用

- **当** 计时器被暂停
- **则** 跳过和重置轮次项应当保持启用

#### 场景：重置后跳过和重置轮次被禁用

- **当** 计时器被重置为空闲
- **则** 跳过和重置轮次项应当变为禁用

### 需求：切换项调度正确的计时器命令

点击切换项应当调用 `timer.toggle()`，如果空闲则启动计时器，如果运行则暂停，如果已暂停则恢复。

#### 场景：切换启动空闲计时器

- **当** 计时器空闲且用户点击切换项
- **则** 计时器应当开始运行

#### 场景：切换暂停运行中的计时器

- **当** 计时器运行中且用户点击切换项
- **则** 计时器应当暂停

#### 场景：切换恢复已暂停的计时器

- **当** 计时器已暂停且用户点击切换项
- **则** 计时器应当恢复运行

### 需求：跳过项推进到下一轮次

点击跳过项应当调用 `timer.skip()`，立即完成当前轮次并推进序列。

#### 场景：跳过推进轮次

- **当** 计时器运行或暂停中且用户点击跳过项
- **则** 当前轮次应当完成，序列应当推进到下一轮次

### 需求：重置轮次项仅重置当前轮次

点击重置轮次项应当调用 `timer.restart_round()`，将当前轮次的已用时间归零，不改变轮次类型或在工作/休息循环中的位置。

#### 场景：重置轮次将已用时间归零

- **当** 计时器运行或暂停中且用户点击重置轮次项
- **则** 当前轮次的已用时间应当归零，计时器应当在同一轮次返回空闲状态

#### 场景：重置轮次保留序列位置

- **当** 用户在工作/休息循环的第 2 轮点击重置轮次项
- **则** 重置后轮次类型和工作轮次编号应当保持不变
